Relecloud Extended es la versión colaborativa y ampliada del proyecto individual desarrollado en la primera parte del curso, orientada a aplicar prácticas profesionales de ingeniería del software dentro de un entorno realista de trabajo. La aplicación, construida con Django y desplegada en Azure App Service, incorpora funcionalidades adicionales como el envío real de correos desde info_request, la gestión de imágenes por destino, un sistema de reviews accesible únicamente a usuarios con compras registradas y la ordenación de destinos según su popularidad. Su desarrollo se organiza mediante ramas breves asociadas a los Paquetes de Trabajo (PT1–PT4), integradas a través de Pull Requests revisados y validados mediante un pipeline CI/CD en Azure DevOps que ejecuta pruebas automáticas y garantiza la correcta integración de cada cambio.

El proyecto sigue un flujo de trabajo basado en criterios de aceptación formulados como Quality Attribute Scenarios, una Definition of Done orientada a la calidad del software y el versionado mediante Git con ramas por feature. El objetivo es reproducir un proceso profesional que combine trazabilidad, pruebas, control de versiones y despliegue continuo.
